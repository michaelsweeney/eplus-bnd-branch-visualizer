bnd·viz
EnergyPlus HVAC topology & timeseries, in the browser.
What it is

EnergyPlus tells you everything about how a building's HVAC is wired and how it runs across the year — but that information is scattered across the .bnd branch report, the model geometry, and gigabytes of timeseries output. bnd·viz pulls those together into one interactive picture you can open in a browser tab.

It parses the EnergyPlus .bnd (Branch Node Details) report into a live loop/branch graph — components are nodes, EnergyPlus nodes are the edges — renders the zone geometry in 3D, and lets you scrub the whole simulation year while temperatures and flows animate across both views. No backend, no install, no license server: it runs entirely client-side.

Features
  • Runs 100% in the browser — open a link, explore a building. No backend or install.
  • Reads EnergyPlus at the source — the .bnd branch-node-details report becomes a live loop/branch graph.
  • 3D zone geometry from epJSON, color-mapped to zone air temperature.
  • Scrub the whole simulation year — play it back and watch zone temps and node flows move.
  • One selection, every view in sync — click a zone in the graph or in 3D and the diagram, model, time-series chart, and inspector all light up together.
  • Brush the chart to set the time — the heatmap and per-branch temperature readouts follow your cursor.
  • Whole-plant overview down to a single air loop — air loops, plant, distribution, and zone equipment are grouped automatically.
  • Loop families auto-classified (air / hot water / chilled water / condenser) from actual operating temperatures, not loop names.
  • SI / IP units, adjustable temperature ramp, dark / light themes.
Use it with your own models

Everything bnd·viz reads is a standard EnergyPlus output — if you can run a simulation, you already have the inputs. Drag these onto the app:

  • eplusout.bnd — the Branch Node Details report, written automatically on every run, for the HVAC topology.
  • epJSON — your model file (or an IDF converted with EnergyPlus), for the 3D zone geometry.
  • A timeseries JSON — node/zone temperatures and mass flows pulled from the eplusout.sql SQLite output, for the annual playback.

The repo includes the small Node scripts used to extract the timeseries JSON from a simulation, so you can wire it to your own runs.

Built with
bnd·viz is open source and compiles to static files you can host anywhere (the live demo is on Cloudflare Pages). The stack:
  • Cytoscape.js — the branch/loop graph
  • Three.js — the 3D zone view
  • Vite — build & dev server; everything else is vanilla ES modules
